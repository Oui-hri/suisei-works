Web Design / Coding

Word Wolf Online

オンラインで複数人が参加できる、ワードウルフゲームのWebアプリ制作です。

Overview

オンラインで複数人が参加できるワードウルフゲームです。 ルーム作成・参加、待機画面、話し合い、投票、結果表示など、ゲーム進行に必要な画面と動線を制作しました。

Firebase Realtime Databaseを利用し、ルーム情報やプレイヤー情報をリアルタイムで同期できるようにしています。 複数人で同じ進行状況を共有しながら遊べるよう、データの流れや画面遷移を整理して実装しました。

Concept

対面で遊ぶワードウルフを、オンラインでも気軽に楽しめるようにすることを目的に制作しました。 ルームコードを使って参加できる仕組みにし、離れた場所にいるプレイヤー同士でも同じ部屋でゲームを進められる構成にしています。

ゲーム中に迷わないよう、待機・お題確認・話し合い・投票・結果表示の流れを分かりやすく整理しました。 それぞれの画面で必要な情報が伝わるよう、UIの見やすさや操作のしやすさも意識しています。

画面構成

ルーム作成・参加画面からゲームを開始し、待機画面で参加者を確認できるようにしています。 その後、お題確認、話し合い、投票、結果表示へと進む流れにし、ゲームの進行が自然につながるよう構成しました。

ホストと参加者で操作できる内容が異なるため、ホスト判定を行い、ゲーム開始などの操作を制御できるようにしています。 プレイヤー一覧もリアルタイムで反映されるようにし、参加状況が分かりやすくなるよう調整しました。

制作情報

  • 制作区分：チーム制作
  • 制作期間：約2週間（作業時間：約80〜90時間）
  • 担当範囲：Firebase連携 / ルーム作成・参加機能 / プレイヤー情報同期 / ホスト判定 / 画面遷移実装・デザイン / UI調整/レスポンシブ調整（Home画面・画面遷移）
  • 使用技術：HTML / CSS / JavaScript / Firebase
  • 公開形式：GitHub Pages
  • 制作形式：オンラインゲームWebアプリ

制作ポイント

Firebaseを使ってルーム情報やプレイヤー情報を同期し、オンライン上で同じ進行状況を共有できるようにしました。 ルーム作成時には部屋情報を作成し、参加時にはルームコードをもとに該当する部屋へ入れるようにしています。

また、リアルタイムでプレイヤー一覧が更新されるようにすることで、参加状況を確認しやすくしました。 ホスト判定や画面遷移の処理も整理し、複数人で遊ぶゲームとして必要な動きを実装しています。

オンラインゲームとして動作するだけでなく、待機・話し合い・投票・結果表示の流れが分かりやすくなるようにし、 プレイヤーが迷わず進められるUIを目指しました。
